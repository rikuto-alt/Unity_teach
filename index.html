GitHub Desktopを使ったUnity共同開発の基本手順と注意事項

① 自分のブランチでの変更を自分のブランチへ反映させる方法

  1. GitHub Desktopで自分のブランチに切り替えて、Unityで編集する．
  2. GitHub Desktopで、自分のブランチの状態でコミットする。
  3. そのまま Push originとかFetch originを押下．

② 自分のブランチの変更をmainブランチに反映させる方法

  1. GitHub Desktopで、自分のブランチにいる状態で Fetch origin を押す。
  2. 青色のボタン Create pull request が出てくるので押す。
  3. そしたらGitHubのブラウザ画面が開く。
  4. Pull Request のボタンが出てくるので、コメントを書いてから押す。
  5. マージ先が main になっていることを確認してから、Merge Pull Request を押す。
補足：
mainへの変更を自分で確認する場合、GitHub Desktopでブランチをmainに切り替えてからUnityを開くと確認できるはずです。 うまくいかない場合は、GitHub Desktopでmainにいる状態でPushやFetchを押したり、Unityを開き直したりする反映されてるのが確認できるようになる．

③ mainブランチの状態を自分のブランチに持ってくる方法

  1. GitHub Desktopで、mainを反映させたいブランチを選択する。
  2. メニューの Branch → Update from main を押す。
  3. GitHub Desktopで Push origin を押す。
補足1：
編集中でmainにマージしていない自分のブランチに対してこの作業をしても、mainの内容と編集中の内容が統合されるため、基本的に情報が失われることはありません。 ただし、mainと自分のブランチで同じオブジェクトを触っていると、コリジョンが発生する可能性があります。
補足2：
mainからその子への反映は上記のようにUpdate from mainで良い．mainの子をmainの子の子に反映したい場合は，mainの子の子にブランチを切り替えた後， メニューの Branch → Merge into current branch を押下し，mainの子を選択する．これでOK．他も 同じようにやる

基本的な作業の流れ

  • 自分がUnityで作業したい場合は、① → ② の順番で作業する。
  • その後、Slackでこーしろーがやっていたように「〜のでマージしてください」と一言送る。
  • 一行の誰かが「マージしてください」と言った場合は、③を実施する。

注意事項

絶対にmainブランチで作業しないこと．必ずブランチを切ってから作業すること．これマジで注意．
  • 担当する面やオブジェクトだけを触るようにしよう．必ず．絶対．
  • 他の人が編集したものや触っていそうなオブジェクトを触りたい場合は、事前に一言伝える。
  • 同じオブジェクトを複数人で編集すると、コリジョンが発生する可能性がある。

変なことになっても焦らないで！！落ち着いて！！

  • GitHub Desktopで，Historyの中から壊れていない正常だったコミットを探す．
  • そのコミットで右クリ → Create Branch from Commitを押す．
  • これで，正常な状態のコミットからブランチを切ることができる．そのブランチからまたなんか作業したらOK
